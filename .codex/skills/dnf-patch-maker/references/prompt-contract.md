# Prompt 中文稳定结构契约

## 一、职业 Prompt 固定结构

每个职业技能 Prompt 使用以下章节，字段不随主题变化：

````markdown
# 技能显示名

## 职业稳定语义

说明运动、轮廓、阶段、层次、锚点和命中辨识。

## 职业通用 Prompt

```text
可直接组合的主题无关英文 Prompt
```

## 源资源约束

说明人物/特效/Cut-in/武器边界以及必须继承的源帧语义。

## 阶段验收

说明起手、循环、命中、爆发、收尾或常驻阶段要求。
````

职业 Prompt 不写主题色、主题材质、历史补丁名或猜测的 NPK/IMG 路径。

## 二、主题 Prompt 固定结构

每个主题技能 Prompt 使用以下章节：

````markdown
# 技能显示名 - 主题名

## 职业基础

引用职业根同名 Prompt。

## 主题增量 Prompt

```text
可追加到职业 Prompt 的英文视觉增量
```

## 具体变化

说明色板、材质、粒子、光线和层次如何变化。

## 主题验收

说明该技能在本主题下必须可见的阶段与辨识条件。

## 主题排除

只列本主题不允许的视觉语言，不删除源资源合法内容。
````

主题 Prompt 不写 NPK/IMG 映射，不扩大 manifest，也不统一强制透明、无角色、背景或固定画布。

## 三、执行路由门禁

- 逐技能 Prompt 只能在 manifest/inventory 已核验技术资源与显示名映射后进入具体资源构建；文件同名、中文翻译、Replay 名称或设计稿标题不能建立映射。
- 显示名映射未核验时，不加载猜测的逐技能 Prompt。只能组合源帧语义、职业/主题规则中与具体名称无关的共同约束，以及 manifest 独立授权的允许变化。
- 合成顺序固定为“源帧几何与分类 -> 已核验职业语义 -> 已核验主题增量”。任何 Prompt 都不能新增 IMG、帧、人物层、武器、Cut-in、阶段或部署权限。
- Negative Prompt 只约束新生成的视觉增量，不删除源资源合法的人物、背景、UI、文字、黑色 RGB、透明占位或加色层。
- AI 输出标记为概念素材；回灌前仍须在 Photoshop CC 2018 中按源几何、时序、alpha 和构图适配。
- 修改职业或主题 Prompt 后运行 `tools/Test-DnfPromptTree.ps1`；发布前由 `tools/Test-DnfProjectGate.ps1` 复核全部职业/主题 Prompt 树。

## 四、索引固定结构

职业与主题 `prompts/README.md` 均使用：

1. `职责`
2. `加载顺序`
3. `稳定结构`
4. `当前文件`
5. `覆盖状态`

## 五、合成与回灌

- 英文代码块用于图像模型或美术概念组合；中文章节用于稳定路由和验收。
- 概念画布与负面词必须注明适用条件。
- seed、采样器、ControlNet/LoRA、模型哈希、分辨率和批次参数属于版本化 redraw run plan，不写进职业/主题 Prompt 形成不可审计的隐式执行状态。
- 同一动作组可以固定 Prompt 哈希和生成配置来降低漂移，但每帧仍须记录实际参数；共享 seed 或同一 LoRA 不能单独证明时序连续或技能覆盖。
- NPK 回灌永远继承 manifest 和源帧的尺寸、Canvas、偏移、锚点、alpha、帧序与阶段。

# 女气功师樱花主题规则

本文件只定义女气功樱花主题。继承仓库根级二进制/验证规则和上级女气功职业映射规则；不得在这里扩大全职业范围或改写真实资源映射。

## 主题目标

- 风格：柔和、飘逸、粉色樱花粒子、淡金或粉白高光。
- 保留原技能的运动方向、轮廓、强弱层次、时间节奏和命中辨识度。
- 主题化不能把不同技能统一成相同粉色光团；每个技能仍要保留投射、旋转、护罩、声波、爆发、幻兽等职业语义。
- 不自动加入人物、背景、UI、文字或 Cut-in；源帧本来包含这些元素时按职业 manifest 处理。

## 色板与材质

当前程序调色使用的参考色板为：

```text
#000000 -> #3A0D25 -> #C43F73 -> #FFB7C5 -> #FFF5F8
```

- 高亮区优先粉白，主体为樱花粉/玫瑰粉，暗部可保留深紫红层次。
- Alpha 默认继承源帧，不得为了消除黑底而任意清空 alpha。
- 源纹理中的黑色 RGB 可能承担加色混合或强度语义；必须结合 alpha、混合表现和客户端结果判断。
- 花瓣、雾、金边和光尘是可选主题元素，不要求每帧同时出现。

## Prompt 路由

- 职业通用技能语义位于 `../prompts/`；本主题的色板、材质和技能视觉增量位于 `prompts/`。
- 每个主题 Prompt 固定使用“职业基础、主题增量 Prompt、具体变化、主题验收、主题排除”五段中文结构。
- 合成顺序为“源帧语义与几何 + 同名职业 Prompt + 同名樱花主题 Prompt”。
- `prompts/README.md` 是主题 Prompt 索引，不是 NPK/IMG manifest，也不能证明全技能覆盖。
- AI 概念图可按生成器选择 `512x512` 等画布，但实际回灌必须恢复每个源帧自己的尺寸、Canvas、偏移、锚点、帧序和 alpha。
- 透明背景、人物、背景和 Cut-in 均按源帧与职业 manifest 决定，主题 Prompt 不得统一排除。
- AI 素材回灌前必须使用通过项目真实 API 能力探针的本地 Aseprite 进行适配，保存分层 `.aseprite` 工程并导出 runtime PNG；预览导出不能代替逐帧适配。

## 修改范围

- 只修改上级女气功 manifest 明确列入的纯技能特效。
- 默认排除人物本体、服装、脸、武器实体和 Cut-in。
- 对 `character.img`、`characteraction.img` 等模糊命名，必须逐帧证明其为纯特效后再加入。
- 不因“全技能樱花”名称而自动包含未盘点的 NPK 或公共 fighter effect。

## 视觉验收

- 每个源可见帧在输出中仍可见；源全透明/Hidden 帧保持其语义。
- 起手、循环、命中、爆发和收尾阶段均具有连续的樱花主题表现。
- 螺旋念气场、天雷分身步、雷龙、分身和技能针按上级职业规则做专项回归。
- 在黑、白和棋盘背景查看 alpha、粉色轮廓、暗部与拖尾；不得只看单张联系表。
- 检查全画布黑块、不透明黑底、空帧、原样漏改、人物误染和颜色断层。
- 主题像素规则只能用于已声明修改帧，不能把未修改帧或人物层强制调成粉色。

## 基线与产物

- `npk/!!!女气功全技能-樱花粉-第二次修复原版.NPK` 是不可变历史对照基线。
- `npk/!!!女气功全技能-樱花粉.NPK` 是旧契约下的历史工作产物，仍需用户做目标客户端实机验证；活动构建不得覆盖它。
- 两个包包含重复内部 IMG 路径，不得同时部署。
- 活动构建必须使用新的 `RunId` 与版本化输出路径，在临时输出上通过结构、解码、几何、空帧和主题检查后才可发布到工作区；不得更新或覆盖历史产物。
- 当前 NPK 构建本质是程序调色和已核验帧补充；Aseprite 预览导出、Prompt 清单和历史 Photoshop 文件均不能表述为所有技能已逐帧 Aseprite 重绘。
- 活动 Aseprite 输出必须保存在新的 `frames/edited/<run-id>`、`frames/runtime/<run-id>` 或 `frames/preview/<run-id>` 路径并绑定 `validation/<run-id>` 证据；Aseprite 不承担 DDS/BC 或 NPK 封装。

## 文档边界

- 构建命令、产物哈希和当前差异写入 `README.md`。
- 历史漏洞和格式研究过程可写入 `../../docs/thread-handoff-2026-07-11.md`，但新线程执行不依赖该文件。
- 每个新线程重新读取当前职业/主题规则、manifest、两层 Prompt 和最新 release 报告，不沿用对话中的旧路径或哈希。
- 后续建立 manifest 后，技能/NPK/IMG 映射只维护一份，构建器和文档从该事实源读取，避免漂移。

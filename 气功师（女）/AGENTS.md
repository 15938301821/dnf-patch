# 女气功师职业补丁规则

本文件只定义女气功师职业域。所有 NPK/IMG 操作继续遵守仓库根 `AGENTS.md`；职业通用运动、轮廓和阶段 Prompt 位于本目录 `prompts/`，具体配色、材质和改造细节由各主题目录定义。

## 职责边界

- 在这里记录女气功技能、真实源 NPK、内部 IMG、动作阶段、人物层边界、职业通用 Prompt 和职业级验收。
- 不在这里定义樱花、暗黑、雷电、赛博等主题色或主题 Prompt。
- 不复制通用 NPK 编解码、封包和部署流程。
- 不按技能中文名或英文翻译猜资源名。

## 资源事实源

- 最终应以 `manifest.json` 保存经实际 ImagePacks2 inventory 核验的映射。
- 在 manifest 建立前，当前樱花构建范围以 `tools/Build-SakuraNenPatch.cs` 中的 `SkillPacks`、`CommonEffectPaths` 和显式排除集合为临时事实源，不以旧文档中的 `aura_*` 等示例名为准。
- 客户端更新后必须重新盘点；旧版本资源名不能直接沿用。
- manifest 的每项至少包含：技能显示名、源 NPK、内部 IMG、动作阶段、特效/人物分类、include/exclude、期望帧数和来源哈希。

## Prompt 分层

- `prompts/` 只维护不随主题变化的技能运动、轮廓、阶段、层次、锚点和源资源边界。
- 每个职业 Prompt 固定使用“职业稳定语义、职业通用 Prompt、源资源约束、阶段验收”四段中文结构。
- 主题目录的 `prompts/` 读取同名职业 Prompt，只追加该主题的色板、材质、粒子和具体变化。
- 职业 Prompt 不写主题色或历史补丁名；主题 Prompt 不写 NPK/IMG 映射，也不扩大 manifest 范围。
- 固定画布、透明背景、无角色和居中只能作为已分类概念素材的条件，不能成为回灌硬规则。
- Prompt 文件数量不是职业覆盖证明；未核验技能先补 inventory/manifest，再补 Prompt。

## 全技能定义

“全技能”必须由 manifest 覆盖率证明，至少满足：

- 技能专包和公共 fighter effect 中属于女气功的资源均已盘点。
- 每个技能的起手、循环、命中、爆发、收尾和常驻阶段均有明确状态。
- 公共 IMG 只选择属于目标技能的帧或 album，不扩大到其他格斗家资源。
- 人物本体、武器、Cut-in 和纯特效分开分类，不能仅凭 `character.img`、`effect.img` 等文件名判断。
- Prompt、代表帧和联系表不是覆盖证明。

## 人物层边界

- 默认不改人物本体、脸、服装、武器实体和觉醒 Cut-in。
- 只有逐帧确认是纯特效层，并在 manifest 中显式列入后，才允许处理名字含 `character` 或 `action` 的 album。
- 当前已知人物排除基线包含 `sprite/character/fighter/effect/doublelightningdragon/awakebody_nenmaster_0000.img`；客户端变化后仍需重新核验。
- 不把“无角色”设为所有帧的格式要求；它只是默认修改范围。

## 强制回归场景

每次女气功全技能构建都必须单独覆盖并报告：

- 螺旋念气场：起手不可为空，旋转场、循环和收尾均有可见效果。
- 天雷分身步：新主题必须在实际技能阶段可辨识，不能只改到未使用帧或仍显示原样。
- 雷龙、分身、技能针等多 album/多阶段效果：必须通过实包 inventory 找到完整链路，不得按名称猜测。
- 念气环绕、念气罩和公共 fighter effect：检查常驻/循环帧及与其他职业共享资源的误改风险。
- 所有关键技能：检查黑帧、空帧、人物误染、阶段漏改和 LINK 断裂。

用户报告过“螺旋念气场出手为空”“天雷分身步仍是原样”“技能针缺失”和“很多技能变黑”，这些场景在用户确认关闭问题前一直是必测回归项。

## 职业域接口

本目录的 `AGENTS.md`、`manifest.json` 和 `prompts/` 向项目级 `$dnf-patch-maker` 提供 domain brief，至少包括：

- 职业与目标技能集合。
- 精确源 NPK 和内部 IMG 路径。
- 各技能动作阶段及 include/exclude。
- 人物层和特效层边界。
- 允许变化的帧、元数据和像素规则。
- 主题目录及主题验收标准。
- 已知正确基线和必须保持不变的集合。

项目根不再注册女气功单职业 skill。通用 skill 只消费本目录提供的 brief，不反向猜测女气功映射或主题。

以上职业接口必须在每个新线程从当前文件重新加载；不得要求先读取旧对话或 handoff 才能确定职业范围。

## 当前基线

- 已知正确的“第二次修复原版”是职业/主题项目回归基线，不能被后续优化覆盖。
- 当前优化包必须以该基线做 additive diff；共同 IMG/帧应保持元数据和解码 BGRA 一致，只有 manifest 声明的新增或修改内容可以变化。
- 当前文件名、哈希、数量和离线比较结论以 `manifest.json`、`樱花主题/README.md` 和对应 `validation/*/release.json` 为准；`../docs/thread-handoff-2026-07-11.md` 仅作历史证据。

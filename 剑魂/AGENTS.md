# 剑魂职业规则

本文件约束剑魂目录内的职业稳定语义。仓库根规则始终优先；主题目录只能增加外观约束，不能改变本文件的动作、阶段、资源事实门禁或验收边界。

## 职业职责与边界

- 职业层只描述设计稿明确支持的运动方向、轨迹轮廓、阶段关系、视觉层次、锚点语义和命中辨识。
- 技能名称和设计 Prompt 只提供候选语义，不能证明真实资源位置、人物层分类、帧数或客户端覆盖。
- 色板、材质、主题角色、主题武器、主题招式映射、粒子与光线语言必须留在主题层。
- 不根据职业经验补写源文没有出现的动作、次数、技能机制或资源位置。

## 资源事实源

- 真实资源身份、内部路径、IMG 版本、帧索引、帧数、阶段、include/exclude 和人物/特效分类只由经核验的 manifest 或现场 inventory 决定。
- manifest 或 inventory 缺失时，所有具体资源映射、帧范围和覆盖结论均保持待核验。
- 设计稿中的文件名对照、统一前缀、客户端差异和搜索建议不得写成职业事实。
- Prompt 不扩大 manifest；回灌时仍须继承根规则规定的几何、Canvas、偏移、锚点、alpha、Hidden、LINK、图集、旋转、纹理和压缩语义。

## Prompt 分层

加载顺序为：仓库根规则 -> 本职业规则 -> 经核验的 manifest/inventory -> 同名职业 Prompt -> 所选主题规则 -> 同名主题 Prompt。

职业 Prompt 回答技能在任何主题下如何保持动作和阶段可读；主题 Prompt 只回答在这些语义不变时如何改变外观。两层 Prompt 均不能建立资源映射或部署授权。

## 人物、特效、武器与 Cut-in 边界

- 未经 inventory 确认，不把任何技能按名称默认归为纯特效、人物、武器或独立投射物资源。
- 涉及人物位移、姿态、衣摆或武器动作的条目必须保留源人物与武器语义；主题不能凭空新增或删除这些层。
- 轨迹、残影、剑形元素和命中响应不证明存在独立武器或投射物资源。
- 三觉 Cut-in 是设计层明确的 Cut-in 条目；实际资源身份、人物裁切、背景、UI、文字、alpha 和过场阶段仍由 manifest/inventory 决定，不能套用纯特效构图规则。

## 职业验收与回归

- 每个条目的主要运动方向、轨迹轮廓、空间焦点和明确阶段关系必须在主题合成后仍可辨认。
- 多段动作不能合并成单次模糊爆发；起手、行进、命中、爆发、收尾只在源文或 inventory 支持时声明。
- 人物、武器、Cut-in 和纯特效之间不得因主题 Negative Prompt 发生越界删除。
- 具体阶段或分类未核验的条目必须继续标注待 manifest/inventory 核验。

## 覆盖状态

当前 16 个 Prompt 是从设计稿拆出的候选条目，不证明全技能覆盖、真实资源覆盖或客户端兼容。建立覆盖结论前仍需逐项 inventory、manifest 和全帧验证。
